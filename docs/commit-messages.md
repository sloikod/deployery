# Commit Messages

- Format: `<type>: <description>`.
- Lowercase unless name, no period at the end.
- Present tense, imperative mood ("add" not "added")
- Choose the first type that applies when going top to down in the list.

1. `fix` - something was broken and now it works.
2. `feat` - user can now do something they couldn't before.
3. `refactor` - code changed but user sees no difference.
4. `chore` - tests, tooling, config, deps, scripts, docs, comments changed.
5. `wip` - messy, experimental, or in-progress work not cleanly fitting the types above.
