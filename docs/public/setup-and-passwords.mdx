## Current Status

Deployery currently starts `code-server` with `--auth none`.

There is no Deployery login page, no first-run password setup, and no built-in auth flow right now.

Do not expose a Deployery instance directly to the public internet unless you are comfortable with it being open.

## Later

Proper Deployery-owned auth is intentionally deferred until there is an API and database to support it cleanly.
