## Environment variables

Pass these to `docker run` with `-e`:

### `DEPLOYERY_DOCKER_VOLUME_PATH`

**Default:** `/data`

The path inside the container where the data volume is mounted. This must match the target path in your `-v` flag.

```sh
docker run -e DEPLOYERY_DOCKER_VOLUME_PATH=/mydata -v mydata:/mydata deployery
```

Only change this if you have a specific reason to mount your volume somewhere other than `/data`. Most users should leave it at the default.

### `PORT`

**Default:** `8080`

The port Deployery listens on inside the container. The container exposes `8080` by default - use `-p` to map it to any host port you want.

```sh
# Map host port 3000 -> container port 8080 (default)
docker run -p 3000:8080 deployery

# Or change the internal port too
docker run -e PORT=9000 -p 3000:9000 deployery
```
